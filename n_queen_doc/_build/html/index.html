n_queen_dijkstra_skeleton.eight_queen()[source]¶

As written by Dijkstra in A Short Introduction to the Art of Programming, chapter 9, 1971.

Todo

Implement the function, using Dijkstra’s pseudo-code as an inspiration.

Notice that Dijkstra follows the usual pattern of a recursive algorithm:

  1. The generate function is defined as an inner, recursive procedure
  2. The board is initialized
  3. The generate function is called

Follow the pattern, but adapt where appropriate:

  • When Dijkstra’s just updates the vectors, your algorithm should return a list of solutions
  • In Dijktra, \(n\), the number of queens already placed, is an external variable: you are well advised to pass this value as a parameter to the recursive generate function instead (unless you know what the nonlocal keyword means in Python)
Returns:a (sorted) list of solution codes, where each vector of column numbers is encoded as an integer (call the encode_solution procedure).
Return type:list
n_queen_dijkstra_skeleton.encode_solution(qc)[source]¶

Encode a solution as a radix-10 integer. For example, the solution \([0,4,7,5,2,6,1,3]\) in which rows 0 through 7 have queens in columns 0, 4, 7, 5, 2, 6, 1, and 3 respectively, is encoded as integer \(4,752,613\).

Parameters:qc (list) – a vector a column indices, where each each index is the position of the queen for the corresponding row.
Returns:a radix-10 representation of the vector of indices.
Return type:int

n_queen_dijkstra_skeleton.print_solution(qc)[source]¶

Print a solution chessboard on the console. For example, given the first solution for the 4-Queen problem \([1,3,0,2]\), the function prints the chessboard configuration that follows:

X _ _ _
_ _ _ X
X _ _ _
_ _ X _
Parameters:qc (list) – a vector of column indices, whose first element is the queen position in the first row, the second element is the queen position in the second row, and so on.
